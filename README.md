UFC Data Warehouse is a compact data warehousing project built on UFC fight data. The repository combines multiple raw CSV sources through a Python ETL pipeline, standardizes fighter and fight attributes, and produces a clean integrated dataset ready for analytical use. It also includes the SQL code needed to populate a dimensional warehouse and run example OLAP-style analyses.

The core workflow is centered on ETL.py. The script loads three raw datasets, normalizes column names, aligns red and blue corner information, fixes mismatches between sources, enriches fights with event names, derives additional attributes such as age ranges, undefeated status, ranked/champion flags, and physical differences between fighters, and then exports cleaned outputs. The final integrated file is df_common.csv, while d1_post_ETL.csv and d2_post_ETL.csv store intermediate cleaned versions of the two main sources.

On the warehouse side, populate.sql creates a staging table and a star-schema style model with dimensions for date, event, type, finish, and fighter, plus a central fact_fight table. queries.sql contains sample analytical queries focused on topics such as underdog wins, fighter appearances by rank and stance, and how winner-related physical differences vary by fight finish. The two draw.io files document the dimensional model and star schema used by the project.

In short, the repository shows the full path from raw UFC CSV data to a queryable analytical warehouse: source integration, data cleaning, dimensional modeling, warehouse population, and example business-style analysis.
